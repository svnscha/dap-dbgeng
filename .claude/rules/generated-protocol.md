# Generated protocol

Everything in `src/protocol/` is generated by `protgen/protogen.py` from
`protgen/schema/debugAdapterProtocol.json` - treat it as build output, regenerate (`npm run
generate`) rather than hand-editing. It builds as a static library (`dap-dbgeng-protocol`) whose
precompiled header (`dap_service.h`, which includes `protocol.h`) is propagated `PUBLIC` to
everything that links it, so consumers neither `#include` the protocol nor re-parse the large
generated header per translation unit.

Each DAP type becomes a flattened struct with `nlohmann::json` `to_json`/`from_json`; `dap_service`
is a base class that string-dispatches an inbound message to a `handle_<command>_request(const X&)`
virtual (unimplemented requests fall through to a handler that throws, which `dap_server` turns into
an "Unsupported command" error).

The advertised capability set in `dap_server_initialize.cpp` is the source of truth for what the
adapter supports.

To change the generated output, edit the generator (`protgen/protogen.py`) and rerun `npm run
generate` - never edit the headers under `src/protocol/` by hand.
